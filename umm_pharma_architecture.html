4. Key technical decisions

MMM tool

Recommended: Google Meridian — purpose-built for geo-level hierarchical Bayesian MMM, supersedes LightweightMMM, open-source, well-documented.

Alternative: PyMC-Marketing — more flexible if you need custom model structures or already have PyMC expertise in-house.

MTA algorithm

Shapley value for quarterly strategic attribution (theoretically fair, expensive but worth it on representative sample).

Survival / Hazard model for ongoing tactical attribution — fits pharma's "time to Rx" outcome beautifully.

Unified model implementation

PyMC custom model using Meridian or PyMC-Marketing components as inspiration. Non-centered parameterization for stable sampling. NUTS sampler with target_accept=0.95.

Geo experiments

GeoLift (R) — Meta's open-source end-to-end geo-test platform with synthetic control + power calculators.

CausalPy — Python alternative with Bayesian synthetic control, integrates well if your stack is Python-first.

6. Common pitfalls — rank-ordered by frequency

1. Skipping Phase 0 alignment Without a signed measurement charter, every later phase becomes political. KPI ambiguity destroys trust in outputs.
2. Treating MMM and MTA as siloed outputs to "blend" This is exactly what UMM is not. They must be unified at the technical level — a single Bayesian model, not a weighted average of two reports.
3. No HCP master data layer Garbage identity resolution means garbage MTA. Every source system has its own match logic — none is canonical.
4. Ignoring pharma's long lag Rx data has 4–8 wk delay; MMM adstock is 60+ days; MTA window must be 60–180 days. Pretending it doesn't exist biases everything.
5. Showing point estimates without uncertainty Kills trust the first time you're wrong. Show the full posterior distribution or stakeholders will assume false precision.
6. One geo experiment a year is not enough Priors don't update, model rots. Aim for ≥1 experiment per brand per year as a floor.
7. Compliance as afterthought Fix it in Phase 0 or you'll rebuild the warehouse in Phase 6. Every modeling decision must be reproducible and audit-traceable.
